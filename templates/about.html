What is it all about?

As part of the Web and mobile application security course, I created a web application in the form of a blog with many different vulnerabilities. Your task is to understand how attacks on web applications work and how you can use the vulnerabilities you find to

  • extract unsecured user data
  • gain access to other users' accounts
  • prepare a payload harmful to other users that can be placed on this site

and so on. The topic of the blog is vulnerabilities that can be exploited in web applications. It is worthwhile to familiarize yourself with them, as some of them can be found in this application.

Your task is divided into two parts.

Part I

As described above, the application is vulnerable to many different types of attacks. I recommend that you familiarize yourself with the vulnerabilities described on this blog, but also look into other available sources. (My recommendation would be Web Security Academy). Then try to find these vulnerabilities and expose them.

Part II

An additional feature on the blog are hidden flags based on the CTF (Capture-The-Flag) principle. In some places, finding a vulnerability will result in finding a flag (according to the pattern: flag={string}). There are 10 flags.

Good luck!

P.S. Remember that it is not about finding flags, but about understanding selected vulnerabilities, flags are just an addition suggesting that you have found a vulnerability! The most important thing is how you did it and why it was possible!
